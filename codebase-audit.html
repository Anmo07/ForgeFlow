Severity Issue Evidence / Location Suggested fix
HIGH Organization update/delete endpoints have no authentication or membership check. backend/app/organizations/router.py
@router.put('/{org_id}'), @router.delete('/{org_id}'), @router.get('/{org_id}') do not depend on get_current_user or tenant validation. 		Require current_user=Depends(get_current_user) and verify membership/permission before update/delete; for read, restrict org visibility to members.
HIGH Roles endpoints are unauthenticated (create/update/delete + list by org). backend/app/roles/router.py has no auth dependencies. Gate with get_current_tenant + require_permission or at least get_current_user + org membership checks.
HIGH MFA login branch likely breaks response validation (API may 500). backend/app/auth/router.py: @router.post('/login', response_model=TokenResponse) but may return a dict with mfa_required that doesn't match TokenResponse. Change endpoint response model to a union (e.g. Union[TokenResponse, MFAChallengeResponse]) or separate endpoints for MFA start vs verify.
CRITICAL Frontend dependency vulnerabilities (Next.js + PostCSS). Running npm audit --audit-level=high in frontend/ reports a critical Next.js advisory chain and a PostCSS advisory. Upgrade next to a patched version suggested by npm audit fix --force (then re-test).
CRITICAL Backend dependency vulnerabilities (python-jose, starlette). python3 -m pip_audit -r backend/requirements.txt reports vulnerabilities in python-jose==3.3.0 and starlette==0.38.6 (FastAPI dependency). Upgrade to patched versions (e.g. python-jose >= 3.4.0; upgrade FastAPI / Starlette to patched versions and rerun tests).
HIGH Frontend lint does not run (CI quality gate effectively broken). frontend/: npm run lint fails with “Invalid Options: Unknown options: useEslintrc, extensions …”. Align Next + ESLint configuration. Options:
  • Upgrade Next + ensure flat-config support matches.
  • Or remove eslint.config.mjs and use a legacy ESLint config.
MED Inconsistent API calling style: Org Switcher bypasses apiFetch. This can break local dev when backend runs on a different port. frontend/components/ui/org-switcher.tsx uses fetch('/api/organizations/') instead of apiFetch. Use apiFetch('/api/organizations/') or build the URL using NEXT_PUBLIC_API_URL consistently.
MED Backend local dev compatibility issue on newer Python. Installing backend/requirements.txt fails on Python 3.13 when building psycopg2-binary==2.9.9. Pin dev runtime to Python 3.12 (as CI does), or migrate to psycopg (psycopg3) / a version supporting 3.13.
LOW CSRF cookie SameSite policy is inconsistent. backend/app/common/middleware.py sets csrf_token cookie with samesite='lax', while backend/app/auth/router.py sets it to samesite='strict'. Pick one policy and apply consistently.
